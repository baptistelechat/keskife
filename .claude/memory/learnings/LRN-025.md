# LRN-025 — `ChartLegendContent` shadcn : pas de `flex-wrap` par défaut

| Pattern observé                                                                                                                                                                        | Contexte                                                                    | Application future                                                                         |
| -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------ |
| `ChartLegendContent` rend `<div className="flex items-center justify-center gap-4">` sans `flex-wrap` → avec ≥4 items sur mobile, la légende déborde horizontalement hors du container | Fix responsive Stats page (légende PieChart et BarChart sortait de l'écran) | Tout projet shadcn avec chart à légende multiple → toujours passer `className="flex-wrap"` |

## Solution

```tsx
<ChartLegend
  content={<ChartLegendContent className="flex-wrap gap-x-4 gap-y-1" />}
/>
```

Le `className` est mergé via `cn()` dans `ChartLegendContent` — pas besoin de modifier `chart.tsx`.
