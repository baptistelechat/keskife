# LRN-023 — SVG viewBox tight crop : bounding box réel + stroke/2

| Pattern observé                                                                                                                            | Contexte                                                                                                                                                   | Application future                                                                            |
| ------------------------------------------------------------------------------------------------------------------------------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------- |
| Retirer le fond d'un SVG Lucide agrandi (translate+scale) laisse du padding si le viewBox reste carré — le contenu est plus haut que large | Création de `icon-nobg.svg` pour la topbar Keskife : CalendarDays 244×268 px dans un canvas 512×512, viewBox carré → ~16 px de padding horizontal résiduel | Calculer le bounding box exact du contenu et utiliser ses proportions réelles pour le viewBox |

## Formule

Pour un icône Lucide placé via `translate(tx, ty) scale(s)` dans un SVG :

```
xMin_px = tx + (lucide_xMin - stroke/2) × s
xMax_px = tx + (lucide_xMax + stroke/2) × s
yMin_px = ty + (lucide_yMin - stroke/2) × s
yMax_px = ty + (lucide_yMax + stroke/2) × s
```

Puis `viewBox="xMin_px yMin_px (xMax-xMin) (yMax-yMin)"`.

**Exemple CalendarDays** (stroke 1.5, scale 12.5, translate 106) :

- x : 106 + (3-0.75)×12.5 = 134 → 106 + (21+0.75)×12.5 = 378 → width 244
- y : 106 + (2-0.75)×12.5 = 122 → 106 + (22+0.75)×12.5 = 390 → height 268
- `viewBox="132 120 248 272"` (2 px de marge chaque côté)
