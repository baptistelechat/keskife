# LRN-024 — Recharts Pie : `outerRadius` px → débordement mobile

| Pattern observé                                                                                                                                                               | Contexte                                                           | Application future                                                                           |
| ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------ | -------------------------------------------------------------------------------------------- |
| `outerRadius={110}` est absolu. Avec `aspect-video` (16:9), container mobile ~350×197px → `min(w,h)/2 = 98px` → rayon > hauteur disponible → overflow visible hors de la Card | Fix responsive Stats page (PieChart sortait de l'écran sur mobile) | Tout `PieChart` dans un `ChartContainer` shadcn → toujours utiliser un `outerRadius` relatif |

## Solution complète

```tsx
<ChartContainer config={chartConfig} className="min-h-[280px] w-full">
  <PieChart>
    <Pie
      cx="50%"
      cy="45%"     // décale le centre vers le haut pour libérer ~56px pour la légende
      outerRadius="70%"  // relatif à min(width,height)/2 → ~98px sur mobile 350×280
    />
```

- `w-full` sur `ChartContainer` : obligatoire pour contraindre la largeur au parent
- `cy="45%"` : décale le centre légèrement vers le haut pour laisser de la place à la légende en bas
- `outerRadius="70%"` : calcul interne Recharts = `0.7 × min(w,h) / 2`
