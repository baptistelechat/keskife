# LRN-034 — Dialog contrôlé extern : `onOpenChange(true)` jamais appelé

| Pattern observé                                                                                                                                                                                                                                         | Contexte                                                                                                 | Application future                                                                                                                                                              |
| ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Quand `open` est piloté par le parent (`open={editEntry !== null}`), Radix Dialog ne fire jamais `onOpenChange(true)` — ce callback sert uniquement à la fermeture (Escape, overlay click). Initialiser le local state dans `onOpenChange` = dead code. | Implémentation de `EntryEditDialog` : les valeurs de l'entrée ne se pré-remplissaient pas à l'ouverture. | Toujours utiliser `useEffect(() => { if (open && data) { /* sync */ } }, [open, data])` pour initialiser le state local d'une dialog dont `open` est contrôlé depuis le parent. |
