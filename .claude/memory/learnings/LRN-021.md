# LRN-021 — Tailwind v4 : `bg-(--var)` obligatoire pour CSS vars arbitraires

| Pattern observé                                                                 | Contexte                                       | Application future                                                                                 |
| ------------------------------------------------------------------------------- | ---------------------------------------------- | -------------------------------------------------------------------------------------------------- |
| `bg-[--color-bg]` → `background-color: --color-bg` (CSS invalide, sans `var()`) | Indicateur dot `ChartTooltipContent` invisible | Systématiquement remplacer `[--*]` par `(--*)` dans tout composant shadcn v3 porté sur Tailwind v4 |

Syntaxe :

- ❌ Tailwind v3 : `bg-[--color-bg]` → génère `background-color: --color-bg` → propriété CSS invalide, ignorée
- ✅ Tailwind v4 : `bg-(--color-bg)` → génère `background-color: var(--color-bg)` → valide

S'applique aussi à `border-(--color-border)`, `text-(--color-fg)`, etc. Tous les composants shadcn conçus pour Tailwind v3 utilisant ce pattern sont silencieusement cassés en v4. Chercher `[--` dans les composants ui/ après une migration v3→v4.
