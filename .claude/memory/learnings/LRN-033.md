# LRN-033 — `m.div` flex sans `w-full` → enfants perdent leur largeur

| Pattern observé                                                                                                                                                                             | Contexte                                                                                                                        | Application future                                                                                                         |
| ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------- |
| Wrapper `<m.div>` dans un conteneur flex sans `className="w-full"` → l'enfant prend sa taille naturelle, les `w-full` internes deviennent une référence circulaire et le composant rétrécit | `AuthForm` wrappée dans `m.div` pour l'animation d'entrée — la card est passée de pleine largeur (`max-w-sm`) à taille minimale | Toujours ajouter `w-full` + `max-w-*` sur tout `m.div` wrappant un composant qui dépend de `w-full` dans un conteneur flex |

## Fix appliqué

```tsx
// ❌ — card rétrécit
<m.div initial={...} animate={...}>
  <AuthForm />   {/* Card a w-full max-w-sm interne → référence circulaire */}
</m.div>

// ✅ — wrapper porte la contrainte de largeur
<m.div className="w-full max-w-sm" initial={...} animate={...}>
  <AuthForm />   {/* Card réduit à w-full → s'adapte au wrapper */}
</m.div>
```
