# LRN-022 — Cycling tag par formule garantissant couverture uniforme de tous les enums

| Pattern observé                                                                                          | Contexte                                                                                                                 | Application future                                                                                                                       |
| -------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------ | ---------------------------------------------------------------------------------------------------------------------------------------- |
| Parity slot selection (index pair/impair) laisse silencieusement certaines valeurs enum jamais atteintes | `entriesForDay` v1 : `admin` et `formation` absents car leur `tagIdx` ne se déclenchait que dans le slot non sélectionné | Seed/fixtures : préférer `(base + i * step) % N` où step est coprime avec N ; vérifier que chaque valeur enum apparaît au moins une fois |

**Bug initial :** `slots[3]` sélectionné quand `dayIndex % 2 === 0`, mais son `tagIdx` vaut `dayIndex % 2 === 0 ? 0 : 2` → toujours 0 (dev) quand le slot est pris, jamais 2 (admin).

**Fix :** `TAGS[(dayIndex + i * 2) % TAGS.length]` avec i ∈ {0,1,2} — step=2 coprime avec N=5, tous les tags couverts en 5 jours.
