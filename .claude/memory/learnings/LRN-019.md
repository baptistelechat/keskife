# LRN-019 — `size="icon"` + texte → overflow visuel sur les éléments adjacents

| Pattern observé                                                                                                                                                                               | Contexte                                                                                                                  | Application future                                                                                                    |
| --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------- |
| Un `<Button size="icon">` contenant à la fois une icône et du texte déborde sur les éléments flex adjacents — la variante `size="icon"` contraint à 32×32px mais le contenu texte sort du box | Bouton "Installer" PWA dans la top bar : l'utilisateur avait ajouté le label "Installer" à côté de l'icône `<Download />` | Dès qu'un bouton icon porte du texte, passer à `size="sm"` (ou `"default"`) avec `gap-1.5` pour icône + label alignés |
