# LRN-006 — `user_id` absent INSERT Supabase → 403 RLS même avec JWT valide

| Pattern observé                                                                                            | Contexte                                                                                                                      | Application future                                                                                                                            |
| ---------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------- |
| Supabase n'injecte pas `user_id` automatiquement dans les INSERTs même quand l'utilisateur est authentifié | Politique RLS `user_id = auth.uid()` sur la table `entries` ; insert depuis `useEntries.addEntry` sans passer `user_id` → 403 | Toujours passer `user_id: (await supabase.auth.getUser()).data.user?.id` dans chaque INSERT, ou configurer `DEFAULT auth.uid()` côté Postgres |
