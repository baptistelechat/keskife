# LRN-005 — shadcn v4 CSS vars : OKLCH dans `@layer base`, `var()` dans `@theme`

| Pattern observé                                                                                                                                                                                                                                                     | Contexte                                                                                   | Application future                                                                                                                     |
| ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------ | -------------------------------------------------------------------------------------------------------------------------------------- |
| shadcn/ui v4 génère des valeurs OKLCH (`oklch(0.205 0 0)`) dans `@layer base :root`. `@theme` doit les référencer via `var()` (`--color-primary: var(--primary)`), jamais en inline HSL. Mélanger les deux ou utiliser des bare HSL components produit un CSS cassé | Intégration shadcn v4 + Tailwind v4 sans `init` réussi (components.json créé manuellement) | Ne pas copier des snippets shadcn v3 (HSL) dans un projet shadcn v4 (OKLCH) — structure : `@theme { var() }` + `@layer base { oklch }` |
