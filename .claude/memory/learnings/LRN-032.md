# LRN-032 — SVG icône blanc sur fond clair → créer variante `-dark`

| Pattern observé                                                                      | Contexte                                                                                                     | Application future                                                                                                                                             |
| ------------------------------------------------------------------------------------ | ------------------------------------------------------------------------------------------------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `icon-nobg.svg` avec `stroke="#ffffff"` est invisible sur le fond card (blanc cassé) | Logo Keskife affiché dans `AuthForm` — le header foncé l'affichait bien, la card claire le rendait invisible | Pour tout SVG monochrome utilisé sur fonds de couleurs différentes : dupliquer avec `stroke="<couleur primary>"` comme `icon-dark.svg`, garder les deux assets |

## Pattern

```
public/icon-nobg.svg   → stroke="#ffffff"    pour fonds foncés (header, splash)
public/icon-dark.svg   → stroke="#1e2d4a"    pour fonds clairs (card, page auth)
```

Chaque usage choisit la variante selon son fond — pas de filtre CSS, pas de `currentColor` (couleur non accessible depuis SVG externe via `<img>`).
