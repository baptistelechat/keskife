# BDR-011 — Script seed interactif : readline, sans mot de passe en env

| Décision                                                                        | Pourquoi                                                             | Alternatives considérées                                          | Statut |
| ------------------------------------------------------------------------------- | -------------------------------------------------------------------- | ----------------------------------------------------------------- | ------ |
| `scripts/seed.mjs` lancé via `node --env-file=.env.local` + readline interactif | Évite de stocker le mot de passe Supabase en clair dans `.env.local` | `SEED_PASSWORD` en env (rejeté), service role key (trop puissant) | actif  |

`pnpm seed` → demande email (ou `SEED_EMAIL` en env) + mot de passe masqué via `output: null`, puis purge + insère ~45 entrées sur 3 semaines (jours ouvrés, 5 tags, 3 créneaux/jour).
